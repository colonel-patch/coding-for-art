Lesson 7 - Compound Objects

In this lesson we look at another way to connect objects so that they become a new object.
These objects are called compound> objects

Objectives

By the end of the lesson the student should be able to:
  Understand how create a collection of objects and bind them together into a compound object.
which can be manipulated as a single object. Rotate the compound object.and have something happen when a certain angle is reached
Use the 'sleep' function to momentarily pause program execution.

The program

scene.width = 800
scene.height = 600
scene.range=10
scene.title="Figure 1: Compound objects"

hammer_origin=vector(0,0,0)
hammer_base=vector(-2.5,0,0)
anvil_pos=vector(2.5,-2.5,0)

handle = box(pos=hammer_origin, size=vector(1,5,1), color=vector(0.72,0.42,0))

head = box( size=vector(3,1,1), pos=vector(0,2,0), color=color.gray(.6) )

hammer = compound([handle, head])
hammer.axis = vector(0,-1,0)

anvil=sphere(pos=anvil_pos,radius=1 )

direction=-1
da=pi/180

Things to note

The code which does the work in this program is the line hammer = compound([handle, head])
This joins the two obects as one. The objects 'head' and 'handle' are placed in a list ie between the square brackets [ ].
The next line rotates the hammer in the '-y' plane, along the hammer length, so that it appears side on to the user.

 

 

Exercise 1

Copy the program below and do the following:
Rotate the hammer so that it strikes when it 'strikes' the white sphere, the sphere flashes red momentarily.
Hint: VPython has a sleep() function which mometarily pauses the program .eg sleep(1) will pause the program for a second Use the following to rotate the hammer:
hammer.rotate(angle=direction*da, axis=vector(0,0,1), origin=hammer_base)
Remember angles are measured anti-clockwise from zero degrees in the +x direction and you will need to remember the direction in a variable.

Solution to Exercise 1

scene.width = 800
scene.height = 600
scene.range=10
scene.title="Figure 1: Compound Objects"

hammer_origin=vector(0,0,0)
hammer_base=vector(-2.5,0,0)
anvil_pos=vector(2.5,-2.5,0)

handle = box(pos=hammer_origin, size=vector(1,5,1), color=vector(0.72,0.42,0))

head = box( size=vector(3,1,1), pos=vector(0,2,0), color=color.gray(.6) )

hammer = compound([handle, head])
hammer.axis = vector(0,-1,0)
direction=-1
da=pi/180
anvil=sphere(pos=anvil_pos,radius=1 )

while True:
 rate(100)
 if hammer.pos.y<0 or hammer.pos.y>hammer.size.y:
  direction=-1*direction

 if hammer.pos.y<0 and direction>0:
  anvil.color=color.red
  sleep(0.1)
 else:
  anvil.color=color.white

 hammer.rotate(angle=direction*da, axis=vector(0,0,1), origin=hammer_base)

Things to note

We animate the hammer with a 'while True; statement
We create a global variable 'direction' to indicate if the hammer is travelling clockwise or anti-clockwise.
Each time the hammer y coordinates reach zero at +- 0, the direction is switched, so when the hammer is travelling clockwise and the position of y is zero,
then the 'if hammer.pos.y<0 and direction>0' is true and we know we have hit the sphere.
The colour of the sphere is changed to red and the program then 'sleeps' for 1/10 of a second to allow the user to see the 'strike'.
